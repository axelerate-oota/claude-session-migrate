# 【Claude Code】アカウント切り替えで消えたセッション履歴を復活させる方法（macOS）

> Claude Code のデスクトップアプリでアカウントを切り替えたら、それまでの会話（セッション）履歴が一覧から全部消えた——。
> 結論から言うと **データは消えていません。表示されていないだけ** です。この記事では、消えて見える履歴を今のアカウントに「復活」させる方法を、コピペできるスクリプト付きで解説します。

> 🛠 ツール一式はGitHubで公開しています（MIT・無料）：
> **https://github.com/axelerate-oota/claude-session-migrate**
> Claude Code Skill 版とスタンドアロンスクリプト版の両方を同梱しています。

---

## こんな人向け

- Claude Code（Claude デスクトップアプリの Code タブ）を **複数アカウントで使い分けている**
- アカウントを切り替えたら **過去のセッションが一覧から消えた**
- でも **元のアカウントには戻したくない**（今のアカウントのまま履歴を見たい）
- 環境は **macOS**

> ※ いちばん簡単な解決策は「元のアカウントにログインし直す」です。それで良い人はこの記事は不要です。この記事は「アカウントを戻さずに履歴を見たい」人向けの、ちょっと踏み込んだ手順です。

---

## なぜ「消えた」ように見えるのか

調べてみると、Claude Code のデータは2か所に分かれて保存されています。

### ① 会話本体（アカウントに紐づかない）

```
~/.claude/projects/<プロジェクトパスをエンコードした名前>/<セッションID>.jsonl
```

ここに会話のやり取りが `.jsonl` 形式で丸ごと残っています。**ここにはアカウント情報が一切含まれていません。** だから「どのアカウントの履歴か」はこのファイルからは分かりません。

### ② GUIの一覧用インデックス（アカウントに紐づく）

```
~/Library/Application Support/Claude/claude-code-sessions/<アカウントUUID>/<組織UUID>/local_xxx.json
```

実は、アプリの一覧表示は **このインデックスファイルだけ** を見ています。中身はこんな感じ:

```json
{
  "sessionId": "local_xxxxxxxx...",
  "cliSessionId": "75ef0f1f-....",   ← ①の .jsonl ファイル名に対応
  "cwd": "/Users/you/workspace",
  "title": "【SEO】〇〇案件",
  "lastActivityAt": 1782525964445,
  "model": "claude-opus-4-8"
}
```

つまり仕組みはこうです:

- **会話本体（①）は全アカウント共通の場所に残り続ける**
- **一覧（②）はアカウントUUIDのフォルダごとに分かれている**
- アカウントを切り替える＝別のUUIDフォルダを見にいく＝**前のアカウントのインデックスが見えなくなる**

> よくある勘違い:「.jsonl のアカウント情報を書き換えればいい」——これは**できません**。.jsonl にアカウント情報は入っていないからです。書き換えるべきは ② のインデックスの置き場所（どのフォルダにあるか）だけです。

---

## 解決方法：インデックスを今のアカウントのフォルダにコピーする

やることは1つだけ。

**「消えて見えるアカウント」のインデックスファイルを、「今ログイン中のアカウント」のフォルダにコピーする。**

会話本体（①）は共通なので触りません。コピーした瞬間、アプリの一覧に履歴が復活し、開けば中身もそのまま読めます。

---

## 手順（コピペで実行できるスクリプト）

### 0. 【重要】Claude デスクトップアプリを完全に終了する

起動中だと内部データが上書きされ、反映されない・壊れる恐れがあります。必ず終了してください。

### 1. スクリプトを保存して実行

スクリプトはGitHubに置いてあります。clone するか、`standalone/session-migrate.sh` をダウンロードしてください。

```bash
git clone https://github.com/axelerate-oota/claude-session-migrate.git
cd claude-session-migrate
chmod +x standalone/session-migrate.sh
./standalone/session-migrate.sh
```

> **Claude Code をターミナルで使っている人**は、スキル版がさらに簡単です。
> `cp -R skill ~/.claude/skills/session-migrate` で入れて、`claude` を起動し `/session-migrate` と打つだけ。Claude がアプリ終了→コピー→確認案内まで対話で進めてくれます。

実行すると、

1. アプリ終了の確認
2. **検出したアカウント一覧**（件数・最新セッション名つき）を表示
3. 「コピー元（消えて見える方）」と「コピー先（今ログイン中の方）」を番号で選択
4. コピー範囲（直近2日／7日／すべて）を選択
5. **自動でバックアップを取ってから** コピー実行

という流れで進みます。

> どれが「今ログイン中のアカウント」か分からないときは、いったん適当に1つ新しいセッションを作ってからこのスクリプトを動かすと、**件数が増えた／最新セッション名が見覚えのある方**が今のアカウントだと判別できます。

### 2. アプリを起動して確認

Code タブのセッション一覧に履歴が戻っていれば成功です 🎉

## 手動でやる場合（スクリプトを使わない人向け）

フォルダを開いて手でコピーしてもOKです。

```bash
open ~/Library/Application\ Support/Claude/claude-code-sessions/
```

- 中にUUIDのフォルダがいくつかある＝アカウントの数だけある
- 各フォルダ内の `local_*.json` の数＝そのアカウントのセッション数
- **件数が多い方＝履歴が残っている（消えて見える）アカウント**
- その中の `local_*.json` を、**今ログイン中のアカウントのフォルダ（さらに中の組織UUIDフォルダ）にコピー**

コピー前に、念のため対象フォルダを丸ごと複製してバックアップを取りましょう。

---

## うまくいかないとき

- **一覧に出てこない**：アプリ起動時にキャッシュ側で上書きされることがあります。一度アプリを終了 → もう一度コピー、で直ることが多いです。
- **それでも駄目**：素直に元のアカウントにログインし直すのが確実です（＝この手順は使えない環境だった、ということ）。

---

## 注意事項（必ず読んでください）

- これは **非公式な手順** です。Anthropic 公式のサポート外です。
- アプリのアップデートで **フォルダ構造が変わると動かなくなる** 可能性があります（記事公開時点の挙動です）。
- **macOS 専用** です。Windows / Linux はパスが異なります。
- 実行は **自己責任** で。必ずバックアップを取ってから行ってください（スクリプトは自動でバックアップします）。

---

## まとめ

- アカウント切り替えで消えた履歴は **データとしては残っている**
- 会話本体（`.jsonl`）はアカウント非依存、**一覧インデックスだけがアカウント別**
- インデックスを今のアカウントのフォルダに **コピーするだけ** で復活する

同じところでハマっている人の助けになれば幸いです。役に立ったらスキ・フォローしてもらえると励みになります 🙏
