# claude-session-migrate

> **macOS版 Claude Code（デスクトップアプリ）でアカウントを切り替えたら過去のセッション履歴が消えた、を解決するツール。**
> データは消えていません。**表示アカウントが変わっただけ**です。このツールは、消えて見える履歴を今ログイン中のアカウントへ引き継いで復活させます。

Claude Code skill ＋ スタンドアロンスクリプトの2形態で提供します。

---

## なぜ履歴が消えて見えるのか

Claude Code のデータは2か所に分かれています。

| データ | 場所 | アカウント紐付け |
|---|---|---|
| 会話本体 | `~/.claude/projects/<encoded-path>/<sessionId>.jsonl` | **なし**（全アカウント共通） |
| GUI一覧の索引 | `~/Library/Application Support/Claude/claude-code-sessions/<accountUuid>/<orgUuid>/local_*.json` | **あり**（アカウントごとにフォルダ分割） |

アプリの一覧は **索引（②）だけ** を見ています。アカウントを切り替えると別のUUIDフォルダを見にいくため、前のアカウントの履歴が一覧から消えます。**会話本体（①）は共通の場所に残り続けています。**

→ **解決法：索引ファイルを今ログイン中アカウントのフォルダにコピーするだけ。**
会話本体は共通なので、コピーした瞬間に一覧へ復活し、開けば中身もそのまま読めます。

> ❌ よくある誤解が「`.jsonl` のアカウント情報を書き換える」という発想。でも `.jsonl` にアカウント情報は入っていないので不可能です。動かすのは索引の置き場所だけです。

---

## 使い方 A: Claude Code Skill（推奨）

Claude Code を**ターミナルで** (`claude`) 使っている人向け。Claude 自身がアプリ終了→コピー→案内まで実行します。

```bash
# 1) スキルを導入
#   方法1: .skill ファイルを取り込む（リリースから session-migrate.skill を取得）
#   方法2: skill/ 以下を ~/.claude/skills/session-migrate/ にコピー
cp -R skill ~/.claude/skills/session-migrate

# 2) ターミナルで claude を起動して呼ぶ
/session-migrate
```

スキルが、アカウント一覧の提示 → コピー元/先の確認 → 自動バックアップ → コピー → 起動案内まで対話で進めます。

> ⚠️ **デスクトップアプリの Code タブ内ではなく、ターミナルの `claude` で実行してください。** アプリ内で実行するとコピー中にアプリを終了できず（＝自分のセッションが落ちる）、全自動になりません。

---

## 使い方 B: スタンドアロンスクリプト

Claude Code を使っていない人でも、シェルスクリプト単体で実行できます。

```bash
# 1) Claude デスクトップアプリを完全に終了する（重要）
# 2) スクリプトを実行
chmod +x standalone/session-migrate.sh
./standalone/session-migrate.sh
```

対話形式で「コピー元／先のアカウント」「範囲（直近2日／7日／全件）」を選ぶと、自動でバックアップを取ってからコピーします。

---

## 直接エンジンを叩く（上級者向け）

```bash
python3 skill/scripts/migrate.py list                       # アカウント/セッション一覧
python3 skill/scripts/migrate.py copy --src 3 --dst 1        # 3番→1番（直近2日）
python3 skill/scripts/migrate.py copy --src 3 --dst 1 --days 0   # 全件
```

`copy` は実行前にコピー先を `claude-code-sessions-backup-<日時>` へ自動バックアップします。

---

## うまくいかないとき

- **一覧に出ない**：アプリ起動時にキャッシュ側で上書きされることがあります。アプリ終了 →`copy` を再実行で直ることが多いです。
- **それでも駄目**：元のアカウントにログインし直すのが確実です（＝この手順が使えない環境だった、ということ）。

---

## 注意事項

- **非公式な手順です。** Anthropic 公式サポート外です。
- アプリのアップデートで**フォルダ構造が変わると動かなくなる**可能性があります（記事/コード公開時点の挙動）。
- **macOS 専用**です（Windows/Linux はパスが異なります）。
- 実行は**自己責任**で。スクリプトは自動バックアップを取りますが、念のため大事なデータは別途控えを。
- `.jsonl`（会話本体）は **編集しません**。索引のコピーのみ行います。
